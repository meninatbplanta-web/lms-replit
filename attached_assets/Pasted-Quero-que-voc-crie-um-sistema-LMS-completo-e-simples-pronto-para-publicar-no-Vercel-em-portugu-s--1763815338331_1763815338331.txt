Quero que você crie um sistema LMS completo e simples, pronto para publicar no Vercel, em português do Brasil, inspirado visualmente nas telas anexas (treinamento SendFlow): fundo escuro, destaque em amarelo, tipografia limpa, cards grandes, timeline vertical de aulas e layout consistente em todas as telas.

1. Tecnologia e estrutura geral

Crie um projeto fullstack (frontend + backend) em Next.js com TypeScript, pronto para deploy no Vercel sem necessidade de configurações adicionais complexas.

Use uma arquitetura simples:

API interna para CRUD (courses, modules, lessons, users).

Banco de dados simples (pode ser SQLite ou Prisma com SQLite).

Todo o sistema deve estar em pt-BR (labels, botões, textos, mensagens de erro).

2. Entidades principais

Implemente no backend as seguintes entidades:

Curso (Course)

id

título

descrição curta

imagem de capa (URL)

slug

status (ativo/inativo)

ordem de exibição

Módulo (Module)

id

courseId

título (ex: “Primeiros passos”)

descrição

ordem dentro do curso

Aula (Lesson)

id

moduleId

título (ex: “Comece por aqui”)

descrição

url do vídeo (YouTube, Vimeo ou arquivo mp4)

duração (em minutos ou texto)

data de liberação (releaseAt, tipo datetime)

anexos opcionais (lista de URLs para PDFs, links, etc.)

ordem dentro do módulo

Usuário (User)

id

nome

e-mail

senha (hash)

Progresso do usuário (UserLessonProgress)

userId

lessonId

completed (boolean)

completedAt (datetime)

3. Cursos iniciais já cadastrados

Crie no seed (dados iniciais):

Curso 1: “Minicurso Terapeuta Analista Corporal”

Curso 2: “Formação Terapeuta Analista Corporal”

4. Área do aluno – fluxo e telas
4.1. Tela inicial de Treinamentos

Rota: /treinamentos

Cabeçalho com:

Logo simples no canto esquerdo.

Campo de buscar (ícone de lupa) no topo direito.

Ícone de usuário com nome “Luis” como exemplo.

Conteúdo principal: grid de cards de cursos, mantendo consistência com os anexos:

Fundo escuro

Imagem em preto e branco do celular/laptop (pode ser placeholder)

Título grande: “AULAS EM PORTUGUÊS” / “Minicurso Terapeuta Analista Corporal” etc.

Ao passar o mouse, mostrar linha azul na base do card, igual aos anexos.

Ao clicar em um curso, abre a página interna do curso.

4.2. Página interna do curso (lista de módulos e aulas)

Rota: /curso/[slug]

Layout dividido em duas colunas, como nos anexos:

Coluna esquerda

Card grande do curso com:

Imagem

Botão “Voltar”

Barra de progresso amarela mostrando a % concluída do curso.

Campo “Buscar por título da aula” (input).

Botão largo azul: “Continuar de onde parei”.

Coluna direita

Título do módulo (ex: “Primeiros passos”).

Embaixo, uma timeline vertical com as aulas:

Cada aula é um item com:

Bolinha grande com check amarelo se concluída.

Número da aula (01, 02, 03…).

Título da aula.

Aulas futuras (com data de liberação no futuro) aparecem bloqueadas:

Ícone de cadeado.

Texto “Disponível em [data]”.

Não permitem clique até a data de liberação.

O sistema deve liberar cada aula automaticamente na data e hora cadastradas em releaseAt, sem necessidade de ação manual.

4.3. Página da aula (player)

Rota: /curso/[slug]/aula/[lessonId]

Layout semelhante ao anexo “Comece por aqui”:

Parte central esquerda

Player de vídeo grande (com controles padrão, tempo total etc.).

Título da aula logo abaixo.

Seção “Descrição” com texto e, abaixo, links de anexos (PDFs, links, etc.).

Botão verde grande: “Concluído” para marcar a aula como concluída.

Campo de avaliação com estrelas (1–5).

Seção “Comentários” com:

Campo para digitar comentário.

Lista de comentários (nome do usuário, data, texto).

Coluna direita

Um painel “Caderno de Anotações” com campo para o aluno escrever anotações pessoais da aula.

Abaixo, uma lista vertical de aulas do mesmo módulo, como mini-cards com thumbnail e título (exatamente como na coluna lateral dos anexos).

5. Regras de liberação por data (drip content)

Implemente lógica de liberação de aulas:

Cada aula tem um campo releaseAt.

Se a data atual < releaseAt, a aula:

Aparece bloqueada, com ícone de cadeado.

Não permite abrir o player.

Exibe mensagem: “Esta aula estará disponível em [data formatada]”.

Quando data atual >= releaseAt, a aula é liberada automaticamente:

Ícone muda de “bloqueado” para normal.

Pode ser marcada como concluída.

8. Modo claro/escuro

No modo claro, inverta as cores mantendo o amarelo como cor de destaque.

Player de vídeo ocupando toda a largura.
